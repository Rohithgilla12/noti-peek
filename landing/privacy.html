Privacy, plainly.

Last updated: 2026-04-20
The short version. noti-peek fetches notifications from providers you explicitly connect (GitHub, Linear, Jira, Bitbucket), stores your OAuth tokens in a Cloudflare D1 database scoped to an anonymous device token, and does not sell, share, or analyze your data. Notification bodies live only in your app's memory — never in our database. You can revoke and delete everything from Settings → Disconnect.

1. What we store

Field Where Why
device_token D1 (server), local store (app) Anonymous identifier so your device can fetch its own notifications. No email, no name.
OAuth access & refresh tokens D1, per-provider row Needed to call GitHub/Linear/Jira/Bitbucket on your behalf.
Provider account name / avatar URL D1, per-provider row So the app can show which GitHub account is connected.
OAuth state nonces D1, auto-deleted after ~10 min CSRF protection during sign-in.
Rate-limit counters D1, rolling window So one runaway device doesn't burn the shared quota.

2. What we do NOT store

  • Your name, email, or any provider account data beyond display name + avatar URL.
  • Notification bodies, issue titles, PR diffs, or comment text — those pass through the Worker at request time and are never persisted.
  • Analytics events tied to you personally. No behavioral profile.
  • Your IP address beyond what Cloudflare's edge normally logs for abuse prevention.
  • Any data from providers you haven't explicitly connected.

3. How data flows

When the app asks for notifications, the flow is:

  1. App sends its device_token to api.notipeek.dev.
  2. Worker looks up which providers that device has connected and fetches from each provider's API, in parallel, using your access token.
  3. Results are merged, optionally bundled, and returned to the app.
  4. Nothing is written back to D1 except bookkeeping: rate-limit counters and (if a token was refreshed) the new token.

Requests to api.notipeek.dev are served by Cloudflare Workers, which may log request metadata for abuse prevention. See Cloudflare's privacy policy for their retention.

4. Telemetry & crash reports

The desktop app can optionally send anonymous crash reports via Sentry, scrubbed of notification content. This is off by default in release builds and on-by-default only in explicitly labelled beta builds; you can toggle it in Settings → Diagnostics at any time. We never log provider names, titles, bodies, URLs, or user identifiers in telemetry — only stack traces and app version.

5. Deletion

In the app, Settings → Disconnect on any provider revokes the access token with the provider (where their API supports it) and deletes the corresponding D1 row. Settings → Delete my account deletes the device_token and every associated connection row. After deletion, the server cannot re-identify you; the data is gone.

If you can't reach the app (lost device, etc.), email privacy@notipeek.dev with your device token and we will purge it on request.

6. Third parties

Two categories of third parties receive data:

  • The providers you connect (GitHub, Linear, Jira, Bitbucket). These receive your API calls because they have to — that is the point. Review their policies for their handling.
  • Infrastructure: Cloudflare (Workers + D1) for the API and database, GitHub Releases for app downloads, and optionally Sentry for crash reports if you opt in.

We do not share data with advertisers, data brokers, or analytics vendors. We do not sell data, ever.

7. Security

OAuth tokens in D1 are transmitted only over TLS and scoped to a single device token. A stolen device token lets someone read your notifications but not reset your device-token-to-provider binding without your OAuth consent. We recommend revoking in-app if you suspect any compromise; this invalidates the tokens with the provider directly.

8. Changes to this policy

Material changes will be announced in-app on next launch and in the release notes on GitHub. Dates above reflect the last substantive edit.
